## 1. История и развитие ОС GNU/Linux

### 1. Проект GNU
- Проект GNU (GNU's Not Unix!) был инициирован Ричардом Столлманом в 1983 году для создания полностью свободной/открытой Unix-подобной операционной системы.
- Были созданы многие компоненты ОС: компиляторы, библиотеки, утилиты, оболочки (например, bash), но долго не было свободного ядра.

### 2. Появление ядра Linux
- В 1991 году Линус Торвальдс, студент из Финляндии, начал разработку своего собственного ядра, которое назвал Linux.
- Linux был выпущен как открытое ПО, и вскоре к нему подключилось сообщество разработчиков по всему миру.
- Ядро Linux слилось с уже готовыми компонентами GNU, образовав полноценную ОС GNU/Linux.

### 3. Развитие GNU/Linux
- Впервые GNU/Linux начали использовать энтузиасты и университеты, затем появились дистрибутивы (Slackware, Debian, Red Hat и др.), которые упростили установку и поддержку системы.
- Сегодня Linux работает на серверах, суперкомпьютерах, встраиваемых и мобильных системах (например, Android), а также на настольных ПК.
- Разработка ведётся открыто, с участием тысяч программистов и множества компаний.

## Single UNIX Specification и POSIX

### POSIX (Portable Operating System Interface)

- POSIX — это набор стандартов (IEEE 1003, ISO/IEC 9945), определяющих совместимые интерфейсы операционных систем, главным образом на базе Unix.
- Охватывает системные вызовы ядра, стандартные библиотеки, командные оболочки, утилиты.
- Главная цель: обеспечить переносимость программного обеспечения между разными Unix-подобными ОС (и даже не-UNIX).

**Примеры поддерживаемых стандартов POSIX:**
  - syscalls для процессов, потоков, файлов, сигналов и т.д.
  - Shell-команды (sh, ls, cp и др.)
  - API для многопоточности (pthreads)

### Single UNIX Specification (SUS)

- Single UNIX Specification — расширенный отраслевой стандарт для Unix-подобных систем, поддерживаемый организацией The Open Group.
- Включает стандарты POSIX, а также дополнительные требования к возможностям и совместимости.
- Сертификация по SUS даёт право ОС официально называться "UNIX" (например, AIX, HP-UX, Solaris).

---

### Отличия
POSIX — это основной набор требований, минимальная совместимость (базовые API и утилиты для совместной работы программ). (Например макось - совсемстима с posix, но не с sus)
SUS — это более полный и строгий стандарт, включает POSIX и добавляет требования (например, строгое поведение командной оболочки, дополнительные утилиты, формат документации) (solaris совместима с sus и posix)
