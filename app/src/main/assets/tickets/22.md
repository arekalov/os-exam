## Процесс: определение

**Процесс** — у процесса много определений:
- экзмемпляр выполняющейся программы
- еденица активности ОС, характеризуемая последовательностью команд, текущим состоянием и связанными ресурсами.

---
## Характеристики процесса в момент выполнения

1. **Идентификатор**
   - Уникальный идентификатор, связанный с этим процессом, чтобы отличать его от других процессов в системе (PID).

2. **Состояние**
   - Характеризует текущий этап жизненного цикла процесса — например, выполнение, ожидание, готовность.

3. **Приоритет**
   - Показывает относительную важность процесса. Определяет, сколько процессорного времени будет предоставляться процессу по сравнению с другими процессами.

4. **Программный счетчик (PC)**
   - Адрес следующей выполняемой команды программы.

5. **Указатели памяти**
   - Включают адреса на программный код, данные, связанные с процессом, и на возможные блоки памяти, используемые совместно с другими процессами (например, при использовании общей памяти).

6. **Данные контекста**
   - Содержат значения регистров процессора и другие аппаратные параметры, необходимые для правильного возобновления процесса после его приостановки.

7. **Информация о состоянии ввода-вывода**
   - Описывает внешние запросы на ввод-вывод, устройства, назначенные процессу, список файлов и устройств, используемых процессом.

8. **Учётная информация**
   - Включает данные о времени процессорной работы, времени выполнения, учётных записях, используемых ресурсах и прочую статистику, важную для планирования и безопасности.

## Состояния процесса

Обычно процесс в ОС может находиться в одном из следующих состояний:

1. **Новый:**  
    Только что созданный процесс, который еще не помеещн ос в пул выполнимых процессов.
2. **Готов к выполнению:**  
   Процесс, который может быть запущен, как только для этого представится возможность.
3. **Выполняющийся**  
   Процесс, который выполняется в текущий момент времени.
4. **Блокированный**  
   Процесс ,который не может выоплняться до тех порпока не произойдет некоторое событие, напимер завершение io.
5. **Заверщающийся**  
   Процесс, удаленный ос из пула выполнимых процессов из-за завершения работы или ошибки.

## Разделение ресурсов между процессами

Для эффективной работы многозадачной ОС ресурсы системы (процессорное время, память, устройства ввода-вывода и др.) разделяются между процессами по определённым правилам:

- **Память:**  
  Каждый процесс получает своё виртуальное адресное пространство. Совместное использование памяти возможно через механизмы IPC (shared memory).
- **Процессор:**  
  ОС планирует выполнение процессов согласно алгоритму (Round Robin, приоритеты и пр.), выделяя каждому квант времени.
- **Файлы и устройства:**  
  Система управляет таблицами открытых файлов/устройств для изоляции доступа.
- **Сетевые ресурсы:**  
  Операционная система отслеживает сетевые соединения и распределяет их между процессами.
- **Синхронизация:**  
  Для работы с разделяемыми ресурсами используются семафоры, мьютексы и другие механизмы.
