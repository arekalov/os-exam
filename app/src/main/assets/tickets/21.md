## Основные сервисы и функции Windows

**1. Windows API functions**
- Основной набор программных интерфейсов для приложений.
- Примеры функций:  
  - `CreateProcess` — создание новых процессов  
  - `CreateFile` — работа с файлами и устройствами

**2. System calls (Native System Services)**
- Низкоуровневые вызовы, напрямую обращающиеся к ядру системы.
- Пример:  
  - `NtCreateUserProcess` — системный вызов для создания процесса

**3. Kernel support functions**
- Внутренние функции ядра для управления памятью, ресурсами и др.
- Пример:  
  - `ExAllocatePoolWithTag` — выделение памяти из пула с тегированием

**4. Windows services**
- Долгоживущие фоновые процессы, управляемые через Service Control Manager.
- Отвечают за работу системного ПО, безопасности, сети, обновлений и других задач.

**5. Dynamic link libraries (DLL)**
- Динамические библиотеки, которые подгружаются во время выполнения программ.
- Примеры:  
  - `msvcrt.dll` — Microsoft C Runtime Library  
  - `kernel32.dll` — базовые функции ОС

## Другие важные компоненты системы

**1. Гипервизор Hyper-V**
- Встроенная технология виртуализации.
- Позволяет запускать гостевые операционные системы.
- Используется также субсистемами Device Guard, Hyper Guard, Credentials Guard, Application Guard и др.

**2. Firmware (в том числе для устройств)**
- Микропрограммы для аппаратных средств, обеспечивающие их работу и взаимодействие с ОС.

**3. Terminal Servers**
- Служба, предоставляющая удалённый доступ к рабочему столу и приложениям (Remote Desktop/Terminal Services).

**4. Объекты и безопасность**
- Система управления объектами (файлы, потоки, процессы и др.) с продвинутыми механизмами контроля доступа и разграничения прав.

**5. Registry (Реестр)**
- Иерархическая база данных конфигурации всей системы и программ.
- Используется для хранения настроек ОС, драйверов и приложений.

**6. Оснастки (Microsoft Management Console, MMC Snap-ins)**
- Конфигурируемые инструменты управления системой: управление дисками, пользователями, службами и сетевыми настройками.
