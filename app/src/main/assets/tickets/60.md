## Предпосылки
* Потребность в постоянном увелчении производительности и дисковой памяти
* Производительности вторичных хранилищ (диски) растет медленнее первичных (CPU, ОЗУ)
  
**JBOD** (Just a Bunch of Disks) - массив независимых и параллелььно работающих HDD, по которым единое логическое пространство распределено последовательно. Из плюсов провышение скорости IO за счет обращение CPU к разным дисками, однако чем больше дисков при такой организации, тем меньше надежности из-за увеличения вероятности отказа диска

## RAID

**RAID** (Redundant Array of Independent Disks) - набор дисков, объединенных в один логический том. 

##№ Особенности:
* Данные распределены по всем дискам
* Могут быть нужны дополнительные диски для контрольной информации, нужной для восстановления данных при отказе
* Дублирование данных 
* Запрос может выполняться параллельно, если блок распределен по нескольким дискам
* Требуются мощные адаптеры? шины подключения к адаптерам и шины IO (PCI, PIe)

Таким образом, один блок на нескольrих дисках и параллельность - ведут к увеличению производительности

RAID 0, 1 (10, 0+1), 2, 3, 4, 5, 6 - различные конфигурации, отличающие по скорость/отказоустойчивости
